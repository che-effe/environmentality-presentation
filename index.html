Designing and developing the application

The first step in actually building the application was finding the right tools.

The right tools for the job

I needed to do some research on what technologies would be available that would best suit my needs for developing the app.

I decided to use Ruby on rails, even though I had no experience with it.

I had to deal with a learning curve at the beginning but it proved to be the best tool for the job.

For secure, bulk storage of image files, I used Amazon s3 which proved to be an easy to implement tool to store a large amount of images securely.

For hosting and deployment of the application, I choose Heroku. Heroku also holds onto my Postgresql database. Deploying builds of the application through heroku has been very reliable, though it can start to get expensive when you start scaling things up. I may have to find a different deployment strategy in the near future, but it has been great for getting the app off the ground.

For versioning control and deployments through heroku, I used Git, which has proven to be an invaluable tool, and I feel should really be taught in the program though it is not.

the applications foundations

10 different technologies!

Most of the logic for creating users, sessions and most everything else pertaining to the database and flow of the application were handled by using ruby

All of the views of the application are written in erb, which is a ruby templating language that builds the final html that the server delivers to the client.

All of my styling was written in css. and there was some lower level logic that I handled in Javascript.

In the end, I ended up using somewhere around 10 different technologies to build the application
